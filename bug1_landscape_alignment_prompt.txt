任务：修复横屏状态下首页元素未整齐对齐的 Bug

背景
这是一个母亲节礼物 App（Kotlin + Jetpack Compose）。项目路径：app/src/main/java/com/example/happy_mothers_day/

核心架构：
- AudioManager (audio/AudioManager.kt)：Kotlin object 单例，管理 MediaPlayer 播放状态，暴露 isPlaying, duration, currentPositionMs, currentSourcePath 等全局状态
- HomeScreen (ui/screens/HomeScreen.kt)：首页，含 MiniPlayer（底部浮动迷你播放条）、NFC 状态卡片、直接播放按钮。横竖屏分别走 LandscapeLayout / PortraitLayout
- PlayerScreen (ui/screens/PlayerScreen.kt)：全屏播放界面，含黑胶唱片动画和完整进度条
- MainActivity.kt / MainApp：Navigation 路由，首页→播放页传递 autoPlay 和 uri 参数

Bug 描述
横屏状态下，首页中的 MiniPlayer（播放进度条）、NFC 提示卡片、"直接播放音频"按钮三者纵向/横向布局不协调：
1. MiniPlayer 浮动在底部覆盖内容，而 NFC 卡片和按钮在屏幕中间垂直居中，三者视觉上参差不齐
2. 当 MiniPlayer 可见时，它遮挡了 LandscapeLayout 底部的内容区域

根因
文件：ui/screens/HomeScreen.kt

1. LandscapeLayout（第361行）没有像 PortraitLayout（第333行）那样为 MiniPlayer 留出底部间距。PortraitLayout 有 padding(bottom = if (miniVisible) 110.dp else 0.dp)，而 LandscapeLayout 完全没有这段逻辑。
2. MiniPlayer 使用 Box + align(Alignment.BottomCenter) 浮动在父容器底部，与 LandscapeLayout 的 Row 内容产生重叠。

修改要求
1. 给 LandscapeLayout 添加底部 padding，值与 PortraitLayout 保持一致：padding(bottom = if (miniVisible) 110.dp else 0.dp)。需要将 miniVisible 作为参数传入 LandscapeLayout。
2. 确保横屏时 MiniPlayer 的 compact 模式与 LandscapeLayout 的内容区在视觉上协调——MiniPlayer 不应遮挡 NFC 卡片或按钮。可将 LandscapeLayout 的内容整体上移，为 MiniPlayer 预留底部空间。
3. 可考虑在横屏时将 MiniPlayer 宽度限制为与 LandscapeLayout 的 Row 等宽，使其视觉上更整齐，而不是撑满整个屏幕宽度。

注意事项
- 不要修改 UI 主题颜色、字体、动画等与布局对齐无关的内容
- 保持横竖屏各自的布局策略：竖屏是 Column 滚动布局，横屏是 Row 并排布局
- 保持 MiniPlayer 在竖屏时的现有表现不变
- 横屏 MiniPlayer 仍应使用 compact 模式（已有参数 compact = isLandscape）
